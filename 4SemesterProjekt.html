Kickoff · oprettelse af BMC og begyndelse af planlægning

Mål for dagen

  • Udarbejde BMC
  • Planlægge research inden for Backendudvikling & API-design samt Cybersikkerhed

Backendudvikling & API-design – research

  • Azure Monitor API og Azure Resource Graph
    • Hente CPU-, RAM-, disk- og cost-metrics programmatisk
    • Filtrere på inaktive eller overbelastede VM’er direkte i forespørgslen
    • Azure Monitor REST API dokumentation
  • Azure Functions
    • Opret HTTP-trigger function til at modtage kald og returnere metrics
    • Opsæt Timer-trigger til automatisk indsamling
  • API-struktur
    • Designe endpoints, så frontenden nemt kan hente data
    • Returnere både realtidsdata og historiske data

Cybersikkerhed – research

  • Microsoft Authentication (Azure AD)
    • Opsætte app registration i Azure AD
    • Forstå OAuth2 Authorization Code flow og integration med API’et
  • Bearer token validering i Azure Functions
    • Sikre at API’et kræver gyldigt token
    • Implementere rollebaseret adgang (RBAC) – fx kun driftsfolk kan stoppe en VM
  • Zero Trust-principper
    • “Verify explicitly” → alle API-kald kræver godkendelse
    • “Least privilege” → brugere får kun de nødvendige rettigheder

Anbefalet rækkefølge (POC)

  1. Start med Azure Monitor API → lav et simpelt script eller function, der henter CPU-forbrug for alle VM’er
  2. Sæt dig ind i Azure AD app registration → lav et testlogin og hent et token
  3. Kombinér de to → lav en Azure Function, der kun returnerer metrics, hvis token er gyldigt
